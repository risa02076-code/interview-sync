# PRD — 인터뷰싱크 (Interview Sync)

면접 일정 자동 매칭 서비스

## 1. 문제 정의

채용팀 리크루터는 패널 면접(면접관 2인 이상) 일정을 잡을 때마다 면접관 전원의 가능 시간을 취합하고, 후보자에게 안내하고, 회의실을 확인하고, 확정 후 리마인드를 보내는 과정을 매번 수작업으로 반복한다. 특히 확정된 면접의 면접관 일정이 바뀌면 이 과정을 처음부터 다시 밟아야 해서, 채용 규모가 커질수록 리크루터의 반복 업무 시간이 비례해서 늘어난다.

이 서비스는 **후보자 희망시간 · 면접관 가능시간 · 회의실 가능시간을 자동으로 대조**해 면접 일정을 확정하고, 확정 후 조건이 바뀌면 **자동으로 재조율**하며, 매칭이 불가능한 경우에만 리크루터가 직접 개입하도록 만들어 반복 업무 시간을 줄이는 것을 목표로 한다.

## 2. 타겟 유저

- **주 사용자**: 채용팀 리크루터 — 후보자를 등록하고 매칭 결과를 확인·관리한다.
- **간접 사용자**: 면접관 · 후보자 — 직접 이 화면을 조작하지는 않지만, 확정된 일정과 회의실의 영향을 받는다. (알림 발송은 이번 MVP 범위 밖)

## 3. 핵심 기능 (MVP)

| # | 기능 | 설명 |
|---|------|------|
| 1 | 후보자 등록 | 이름 · 지원 직무 · 면접 패널(2인 이상) · 희망 시간대를 입력해 새 면접 케이스를 만든다 |
| 2 | 자동 매칭 | 등록 즉시 패널 전원과 회의실이 모두 비어있는 시간을 찾아 자동으로 확정한다 |
| 3 | 매칭 실패 시 에스컬레이션 | 공통 가능 시간이 없으면 사유와 함께 "리크루터 확인 필요" 상태로 표시한다 |
| 4 | 조율 대시보드 | 전체 면접 케이스를 상태(확정 / 에스컬레이션 / 응답대기)별로 한눈에 확인한다 |
| 5 | 자동 재조율 | 확정된 면접의 면접관 일정이 바뀌면, 전체 시간대를 다시 탐색해 대체 시간을 자동으로 확정한다 |
| 6 | 케이스 삭제 | 취소된 면접 건을 목록에서 제거한다 |

## 4. 화면 구성

| 화면 | 경로 | 설명 |
|------|------|------|
| 조율 대시보드 (목록) | `/interviews` | 전체 면접 케이스를 카드 목록으로 표시, 상태별 필터, 상세 화면 링크, 삭제 |
| 후보자 등록 (입력 폼) | `/interviews/new` | 이름 · 직무 · 패널 선택 · 희망시간 선택 후 제출 → 자동 매칭 실행 → 대시보드로 이동 |
| 케이스 상세 | `/interviews/[id]` | 확정 정보(시간 · 회의실) 또는 에스컬레이션 사유 표시, "면접관 일정 변경 시뮬레이션"(재조율 트리거) 버튼, 삭제 |

## 5. 데이터 구조

```
interviewers (면접관)
  id            uuid, PK
  name          text
  role          text            -- 예: "백엔드팀장"
  busy_slots    int[]           -- 이미 잡혀있는 슬롯 인덱스 목록

rooms (회의실)
  id            uuid, PK
  name          text
  busy_slots    int[]

interviews (면접 케이스)
  id                 uuid, PK
  candidate_name     text
  position           text
  panel              text[]     -- interviewers.id 참조 목록
  preferred_slots    int[]      -- 후보자가 제출한 희망 슬롯 인덱스
  matched_slot       int, null
  room_id            uuid, FK -> rooms.id, null
  status             text       -- 'confirmed' | 'escalated' | 'pending' | 'rescheduled'
  note               text, null
  created_at         timestamptz, default now()
```

슬롯(slot)은 사전에 정의된 10개의 면접 가능 시간대(예: "7/20(월) 10:00" ~ "7/24(금) 15:00")를 0~9 인덱스로 표현한 것으로, `interviewers.busy_slots` / `rooms.busy_slots` / `interviews.preferred_slots` 는 모두 이 인덱스 체계를 공유한다.
